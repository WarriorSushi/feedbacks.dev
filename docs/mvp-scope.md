# MVP Scope

## MVP objective

Ship a version of `feedbacks.dev` that proves four things:

1. teams can install it quickly
2. feedback quality is high enough to be useful
3. the dashboard helps with triage
4. teams are willing to keep it in their workflow

## Must-have scope

- authentication
- project creation
- project-specific install snippet
- website, React, and Vue install examples
- floating modal widget
- inline widget support
- custom trigger support
- message submission
- automatic context capture
- optional email capture
- inbox view
- status workflow
- detail view
- basic notifications or webhook integration

## Deliberately excluded from first release

- public roadmap boards
- full-blown feedback voting systems
- complex analytics dashboards
- AI-heavy classification as a dependency
- multi-seat enterprise permission models
- native mobile SDKs

## MVP quality bar

- install documentation must be internally consistent
- widget must feel lightweight
- first-run setup must not require a tutorial video
- primary user journey must be testable end to end

## Release phases

### Phase 1

- project creation
- install flow
- widget submission
- inbox

### Phase 2

- richer customization
- tags
- notifications

### Phase 3

- issue-routing integrations
- pattern detection
- reporting improvements
