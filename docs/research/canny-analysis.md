# Canny.io Competitor Analysis

## What Canny Has

### Public Feedback Boards
- **Multiple boards per product** (Feature Requests, Integrations, Languages, etc.)
- **Voting system**: Upvote button on each post with vote count
- **Post creation**: Title + Details + file upload
- **Sorting**: Trending, Top, New
- **Status badges**: Under Review, Planned, In Progress, Complete
- **Anonymous mode**: Users can post anonymously
- **Search**: Full-text search across posts

### Post Detail Page
- **Vote count** with large upvote button
- **Admin responses** with "member badge" — clearly distinguished from user comments
- **Activity Feed** with sort (Newest first)
- **Comments**: Users and admins can comment, with like/reply functionality
- **Status change history**: Shows who changed status and when (e.g., "Sarah Hum marked this post as under review")
- **Voter list**: Shows who voted (avatars + count)
- **Created by**: Attribution with date

### Roadmap View
- **Kanban-style columns**: Planned, In Progress, Complete
- **Cards** showing post title, vote count, status
- **Visual progress tracking** for product direction

### Changelog
- **Separate page** for shipped features/updates
- **Rich content** — markdown/HTML entries with dates
- **Labels/categories**: New, Improved, Fixed
- **Links back to original feedback posts**

### Navigation Structure
- **Left sidebar**: Boards list, Roadmap, Feedback, Changelog links
- **Top bar**: Logo, login/signup, anonymous toggle
- **Clean, minimal design** — lots of white space

### Admin Features (from their marketing)
- Autopilot — AI auto-categorization
- User segmentation
- Jira/GitHub/Slack/Discord integrations
- Custom domains
- Webhooks
- API access
- SSO
- Custom branding

## What feedbacks.dev Has That Canny Doesn't
- **MCP Server / AI Agent API** — unique differentiator
- **Embeddable widget** (Canny requires separate board URL or iframe)
- **Self-hostable / open source**
- **Simpler pricing model**

## Feature Gap (What We Should Add)

### Must Have (This Sprint)
1. **Admin comments on public board** — let owners respond publicly with a "Dev" badge
2. **Activity feed on public board** — show status changes + comments chronologically
3. **Search on public board** — simple text search
4. **Better post detail view** — expand/modal when clicking a feedback item

### Nice to Have (Next Sprint)
1. **Roadmap/Kanban view** — show Planned/In Progress/Done columns
2. **Changelog page** — show shipped features
3. **Multiple boards per project** — different categories
4. **Rich text in admin comments** — markdown support
5. **User avatars** on public board
6. **Email notifications** when status changes on voted items

### Future
1. AI auto-categorization (autopilot equivalent)
2. User segmentation
3. Custom domains for public boards
4. SSO
